Lesson 2: Structuring a Web Page

In the last lesson, we learned about individual HTML tags. Now, let's see how they fit together to create a complete, functional web page. Every HTML document has a standard "boilerplate" structure that acts as its foundation.

The HTML Boilerplate

Think of this as the minimum required code for any webpage. It tells the browser what kind of document it's reading and contains the main sections for your content and for hidden information. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First Web Page</title>
</head>
<body>
    <!-- Your content goes here! -->
</body>
</html>

Key Structural Tags Explained

Let's break down the boilerplate:

  • <!DOCTYPE html>: This is the very first thing on the page. It declares that the document is an HTML5 document.
  • <html>: The root element. All other elements are nested inside this one.
  • <head>: This section contains meta-information about the page that is not displayed directly on the page itself. This includes the page title (which appears in the browser tab), character set information, and links to CSS files.
  • <body>: This is where the magic happens! All visible content—headings, paragraphs, images, links, etc.—goes inside the body tag.

Semantic Tags for Better Structure

Inside the <body>, we can use more descriptive tags to define sections. These are called "semantic tags" because they give meaning to the structure. 

<body>
    <header>
        <h1>My Awesome Website</h1>
        <nav> ... navigation links ... </nav>
    </header>
    
    <main>
        <p>This is the main content of my page.</p>
    </main>
    
    <footer>
        <p>Copyright 2025. All rights reserved.</p>
    </footer>
</body>
  • <header>: Used for introductory content, typically containing a heading and navigation.
  • <main>: Represents the dominant, main content of the page.
  • <footer>: Used for the bottom of the page, usually containing copyright info and contact links.

Using these tags not only helps organize your code but also improves accessibility and search engine optimization (SEO).
