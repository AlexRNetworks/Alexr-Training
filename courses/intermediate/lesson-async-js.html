Lesson 1: Asynchronous JavaScript & Promises

JavaScript is "synchronous" by default, meaning it runs one line of code at a time. If one line takes a long time to complete (like fetching data from a server), it blocks everything else. Asynchronous JavaScript solves this problem.

Promises

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. It's a placeholder for a value you don't have yet. A promise can be in one of three states: pending, fulfilled, or rejected.

Async/Await

async and await are modern keywords that make working with promises much easier and more readable. An async function always returns a promise. The await keyword can only be used inside an async function, and it tells JavaScript to pause execution until the promise is settled.

// This function simulates fetching data, which takes time
const fetchData = () => {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve("Data has arrived!");
    }, 2000); // Wait 2 seconds
  });
};

async function getData() {
  console.log("Starting to fetch...");
  const data = await fetchData(); // Pause here until the promise resolves
  console.log(data); // This will run after 2 seconds
}

getData();
