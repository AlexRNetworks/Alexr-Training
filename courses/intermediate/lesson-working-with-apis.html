Lesson 2: Working with APIs (The Fetch API)

One of the most powerful features of modern JavaScript is the ability to communicate with servers and get data without reloading the page. We do this by interacting with APIs (Application Programming Interfaces).

The Fetch API

The fetch() API is the modern, built-in way to make web requests. It uses the Promises we learned about in the last lesson, making it perfect for asynchronous operations.

Let's fetch some sample user data from a free public API called JSONPlaceholder.

// The URL of the API endpoint we want to get data from
const apiUrl = 'https://jsonplaceholder.typicode.com/users/1';

// Use fetch to make the request
fetch(apiUrl)
  .then(response => response.json()) // 1. Parse the response as JSON
  .then(user => {                   // 2. Work with the final data
    console.log("User's Name:", user.name);
    console.log("User's Email:", user.email);
  })
  .catch(error => {                 // 3. Handle any errors
    console.error("Failed to fetch data:", error);
  });

// Or, using async/await for cleaner code:
async function getUser() {
  try {
    const response = await fetch(apiUrl);
    const user = await response.json();
    console.log("User's Website:", user.website);
  } catch (error) {
    console.error("Failed to get user:", error);
  }
}

getUser();

This ability to fetch and use data from external sources is fundamental to building rich applications like weather apps, stock tickers, social media feeds, and more.
