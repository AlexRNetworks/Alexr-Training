Lesson 4: React Components, Props, and State

The core building blocks of any React application are components. A component is an independent, reusable piece of the UI. We will focus on function components, which are the modern standard.

Function Components

A function component is literally just a JavaScript function that returns some JSX, which describes what the UI should look like.

function WelcomeMessage() {
  return <h1>Welcome to the course!</h1>;
}

Props: Passing Data to Components

Props (short for "properties") allow you to pass data into your components, much like arguments in a regular function. This makes components reusable and dynamic.

// The component now accepts 'props' as an argument
function WelcomeMessage(props) {
  return <h1>Welcome, {props.name}!</h1>;
}

// When using the component, we pass the 'name' prop
<WelcomeMessage name="Alex" /> 
// This would render: 
Welcome, Alex!

State: A Component's Memory

What if a component needs to remember something that can change over time, like a counter or user input? For this, we use state. The useState hook is the primary way to add state to a function component.

import React, { useState } from 'react';

function Counter() {
  // Declare a new state variable called "count"
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

When the button is clicked, `setCount` is called, React re-renders the `Counter` component with the new `count` value, and the display updates automatically. This concept of state is what makes React so powerful for building interactive UIs.
