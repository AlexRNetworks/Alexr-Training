Lesson 3: Introduction to CSS

So far, our pages have structure, but they lack style. That's where Cascading Style Sheets (CSS) comes in. If HTML is the skeleton of a webpage, CSS is the skin, clothes, and overall appearance. It handles all the colors, fonts, layouts, and visual design.

What is a CSS Rule?

CSS works by defining "rules". A rule selects an HTML element (or elements) and tells the browser how to style it. The basic syntax looks like this: 

selector {
    property: value;
}
  • The selector points to the HTML element you want to style (e.g., h1, p).
  • The property is the style attribute you want to change (e.g., color, font-size).
  • The value is the appearance you want to apply (e.g., blue, 16px).

For example, to make all paragraphs on a page have blue text, the rule would be:

p {
    color: blue;
}

Three Ways to Add CSS

There are three ways to include these CSS rules in your project, but one is highly preferred.

1. External CSS (Best Practice)

You create a separate file with a .css extension (like our own style.css file!) and link to it from your HTML file using a <link> tag inside the <head>. This is the best method because it separates content (HTML) from presentation (CSS) and allows you to style many pages with a single file.

<!-- In your HTML head -->
<link rel="stylesheet" href="style.css">

2. Internal CSS

You can place CSS rules directly inside your HTML file's <head> section by wrapping them in <style> tags. This is useful for page-specific styles (like the ones we use in these lesson files!), but it's not ideal for styles you want to apply across your whole site.

<!-- In your HTML head -->
<style>
    body {
        background-color: lightgray;
    }
</style>

3. Inline CSS

You can apply styles directly to a single HTML element using the style attribute. This method should be used sparingly, as it mixes content and presentation and is hard to manage.

<!-- In your HTML body -->
<h1 style="color: red;">This heading is red</h1>
